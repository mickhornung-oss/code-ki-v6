# V1.1 Dokumentation

## Übersicht

V1.1 erweitert V1 um strukturierte Patch-/Diff-Ausgabe statt nur Freitext.

## Was ist neu?

### Strukturierte Antwort

Die KI liefert jetzt Antworten in einem strukturierten JSON-Format:

```json
{
  "summary": "Kurze Erklärung der Änderung (max. 2 Sätze)",
  "explanation": "Detaillierte Erklärung (optional)",
  "changes": [
    {
      "type": "replace|insert|delete",
      "description": "Kurze Beschreibung der Änderung",
      "old_code": "alter Code (nur bei replace)",
      "new_code": "neuer Code"
    }
  ],
  "risks": ["Optionale Risikohinweise"]
}
```

### Änderungs-Typen

- **replace**: Ersetzt einen Codeblock durch einen neuen
- **insert**: Fügt neuen Code an einer Stelle ein
- **delete**: Entfernt einen Codeblock

### Anzeige in der Extension

Die Extension zeigt jetzt:
- **Zusammenfassung**: Kurze Beschreibung der Änderung
- **Erklärung**: Detaillierte Erklärung (optional)
- **Änderungen**: Liste der Code-Änderungen mit alter und neuer Version
- **Risiken**: Optionale Risikohinweise

## Backend-Änderungen

### Neue Schemas

- `CodeChange`: Repräsentiert eine einzelne Code-Änderung
- `StructuredAnswer`: Repräsentiert die strukturierte Antwort

### Neue Module

- `response_parser.py`: Parst strukturierte JSON-Antworten

### Geänderte Module

- `schemas.py`: Erweitert um strukturierte Antwort-Typen
- `prompting.py`: Angepasst für JSON-Ausgabe
- `service.py`: Integriert Parsing-Logik

## Extension-Änderungen

### Neue UI-Elemente

- Zusammenfassung-Card
- Erklärung-Card
- Änderungen-Card
- Risiken-Card
- Rohantwort-Card (Fallback)

### Neue CSS-Klassen

- `.change-item`: Container für einzelne Änderungen
- `.change-type`: Typ-Label (REPLACE/INSERT/DELETE)
- `.change-desc`: Beschreibung der Änderung
- `.code-block`: Code-Anzeige
- `.code-label`: Label für Code-Blöcke
- `.risk-item`: Risikohinweis

## Fallback-Verhalten

Wenn das Modell kein gültiges JSON liefert:
- Backend behält die Rohantwort
- Extension zeigt die Rohantwort im "Rohantwort"-Card
- Kein Absturz oder Fehler

## Kompatibilität

V1.1 ist vollständig abwärtskompatibel zu V1:
- Die `answer`-Feld bleibt erhalten
- Strukturierte Antwort ist optional (`structured` kann `null` sein)
- Alte Clients können weiterhin die `answer` verwenden

## Nutzung

### Backend starten

```powershell
powershell -File .\scripts\start_backend.ps1
```

### Extension nutzen

1. Extension öffnen: `Code KI: Assistent öffnen`
2. Modus wählen (python_task, rewrite, explain)
3. Prompt eingeben
4. Ausführen
5. Strukturierte Antwort anzeigen

## Beispiele

### Beispiel 1: Python-Funktion verbessern

Prompt:
```
Verbessere diese Funktion:
def add(a,b):
    return a+b
```

Erwartete Antwort:
```json
{
  "summary": "Funktion mit Typ-Hints und Docstring ergänzt",
  "explanation": "Die Funktion wurde um Typ-Hints und einen Docstring erweitert, um die Lesbarkeit zu verbessern.",
  "changes": [
    {
      "type": "replace",
      "description": "Funktionssignatur und Body ersetzen",
      "old_code": "def add(a,b):\n    return a+b",
      "new_code": "def add(a: int, b: int) -> int:\n    """Add two numbers together.\n    \n    Args:\n        a: First number\n        b: Second number\n    \n    Returns:\n        Sum of a and b\n    """\n    return a + b"
    }
  ],
  "risks": []
}
```

### Beispiel 2: Code erklären

Prompt:
```
Erkläre diese Funktion:
def calculate_discount(price, discount_percent):
    return price * (1 - discount_percent / 100)
```

Erwartete Antwort:
```json
{
  "summary": "Berechnet den reduzierten Preis basierend auf einem Rabattprozentsatz",
  "explanation": "Die Funktion nimmt einen Preis und einen Rabattprozentsatz und gibt den reduzierten Preis zurück. Der Rabatt wird als Prozentsatz vom Originalpreis berechnet.",
  "changes": [],
  "risks": []
}
```

## Einschränkungen

V1.1 hat folgende bewusste Einschränkungen:
- Keine automatische Anwendung von Änderungen auf Dateien
- Keine Multi-Datei-Logik
- Keine Git-Integration
- Keine Testautomatik

## Ausblick

Mögliche Erweiterungen für zukünftige Versionen:
- Automatische Anwendung von Änderungen auf Dateien
- Diff-Anzeige mit Zeilennummern
- Multi-Datei-Unterstützung
- Git-Integration
- Testautomatik
